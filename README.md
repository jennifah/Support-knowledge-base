# 📚 Support Knowledge Base — Troubleshooting Runbooks

A structured internal knowledge base built for B2B SaaS support teams.
Covers common integration failures, account issues, and escalation workflows.

Built to mirror real-world support operations at companies like Ashby, Intercom, and Zendesk.

---

## 📁 Structure

```
knowledge_base/
├── docs/
│   ├── integration_troubleshooting.md   ← API & webhook failure guides
│   ├── account_issues.md                ← Login, billing, permissions
│   └── escalation_framework.md          ← When and how to escalate
├── templates/
│   ├── bug_report_template.md           ← Template for filing bugs to Engineering
│   └── customer_response_templates.md   ← Reusable customer-facing responses
└── README.md
```

---

## 💡 Why This Exists

Support teams without documentation repeat the same work every day.
This knowledge base eliminates that — every common issue is documented once,
clearly, so any agent can resolve it independently without escalating.

---

## 🔍 How To Use

1. Search for the issue type in `/docs/`
2. Follow the step-by-step runbook
3. If unresolved after all steps → follow `escalation_framework.md`
4. Use `/templates/` for consistent customer communication
